INTRODUCTION                                           *ambient-introduction*

ambient.nvim plays local music in a game-like ambient style. It can play a
track, wait for a quiet interval, then schedule the next track. It can also run
in continuous playback modes.

The plugin delegates audio playback to `mpv`.

==============================================================================
REQUIREMENTS                                           *ambient-requirements*

Required:

- Neovim
- mpv

Optional:

- ffprobe       Used to read track duration for better progress reporting.
- lualine.nvim  Used for the automatic statusline progress component.

Check your environment with:
>
  :checkhealth ambient
<

==============================================================================
SETUP                                                         *ambient-setup*

Minimal setup:
>
  require("ambient").setup({
    music_dirs = {
      "~/Music/ambient",
    },
    mode = "interval_random",
    volume = 20,
  })
<

Use real directory paths in `music_dirs`. Do not keep angle-bracket placeholder
text in your config because Neovim treats it as a special filename expansion
pattern.

lazy.nvim example:
>
  ---@type AmbientProgressConfig
  local progress = {
    enabled = true,
    layout = {
      width = 42,
    },
    track = {
      width = 18,
      scroll = false,
      scroll_separator = "  ",
    },
    bar = {
      style = "braille",
      width = 10,
    },
  }

  ---@type AmbientConfig
  local opts = {
    music_dirs = {
      "~/Music/ambient",
    },
    mode = "interval_random",
    volume = 20,
    progress = progress,
  }

  return {
    {
      "InubashiriLix/ambient.nvim",
      name = "ambient.nvim",
      main = "ambient",
      event = "VeryLazy",
      cmd = {
        "AmbientStart",
        "AmbientStop",
        "AmbientPause",
        "AmbientTogglePause",
        "AmbientToggleStop",
        "AmbientNext",
        "AmbientPrevious",
        "AmbientPlaylist",
        "AmbientSelectMusic",
        "AmbientStatus",
        "AmbientProgressToggle",
      },
      opts = opts,
      config = function(_, opts)
        require("ambient").setup(opts)
      end,
    },
  }
<

==============================================================================
COMMANDS                                                 *ambient-commands*

                                                        *:AmbientStart*
:AmbientStart
    Start scheduling and playback.

                                                        *:AmbientStop*
:AmbientStop
    Stop scheduling and stop the current track.

                                                        *:AmbientPause*
:AmbientPause
    Pause the current track.

                                                        *:AmbientTogglePause*
:AmbientTogglePause
    Pause while playing, resume while paused, or start playback immediately.
    During a quiet interval, start the next track immediately.

                                                        *:AmbientToggleStop*
:AmbientToggleStop
    Toggle between active playback/scheduling and stopped. A paused track is
    considered active and is stopped.

                                                        *:AmbientNext*
:AmbientNext
    Play the next track immediately.

                                                        *:AmbientPrevious*
:AmbientPrevious
    Play the previous track from playback history.

                                                        *:AmbientPlaylist*
:AmbientPlaylist
    Select the active playlist with |vim.ui.select()|. Current playback or an
    interval is stopped, and the scheduler returns to READY.

                                                        *:AmbientSelectMusic*
:AmbientSelectMusic
    Choose a display sort order, then select and immediately play a track from
    the active playlist with |vim.ui.select()|.

                                                        *:AmbientStatus*
:AmbientStatus
    Show the current scheduler status with |vim.notify()|.

                                                        *:AmbientProgressToggle*
:AmbientProgressToggle
    Show or hide the statusline progress component.

TOGGLE MIGRATION                                  *ambient-toggle-migration*

The :AmbientToggle command and require("ambient").toggle() API were removed.
To preserve the previous start/stop behavior, replace them with
|:AmbientToggleStop| and require("ambient").toggle_start_stop(). For
pause/resume behavior, use |:AmbientTogglePause| or
require("ambient").toggle_pause_resume().

==============================================================================
PLAYBACK MODES                                             *ambient-modes*

interval_random
    Random playback. After each track, wait for a random interval.

interval_sequential
    Sequential playback. After each track, wait for a random interval.

without_interval_random
    Random continuous playback.

without_interval_sequential
    Sequential continuous playback.

intermittently
    Alias for interval_random.

continuous
    Alias for without_interval_random.

The `interval.min_ms` and `interval.max_ms` options only affect interval modes.

==============================================================================
PLAYLISTS                                             *ambient-playlists*

Use multiple directories:
>
  require("ambient").setup({
    music_dirs = {
      "~/Music/ambient",
      "~/Music/piano",
    },
  })
<

Use explicit playlists:
>
  require("ambient").setup({
    playlists = {
      {
        abs_path = "~/Music/ambient",
        ext = { "mp3", "ogg", "flac", "wav" },
        recursive_depth = 4,
        sort_field = "random",
        sort_direction = "asc",
      },
      {
        abs_path = "~/Music/piano",
        ext = { "mp3", "flac" },
        recursive_depth = 2,
        sort_field = "name",
        sort_direction = "asc",
      },
    },
  })
<

The first non-empty configured playlist is active after setup. Use
|:AmbientPlaylist| to select another playlist.

Playlist fields:

abs_path
    Music directory.

ext
    File extensions to scan.

recursive_depth
    Maximum directory recursion depth.

sort_field
    One of: random, name, modify_time, create_time.

sort_direction
    Either asc or desc.

STATUSLINE PROGRESS                                      *ambient-progress*

When lualine.nvim is available, ambient.nvim registers a progress component at
the front of `lualine_x`.

Example:
>
  require("ambient").setup({
    progress = {
      enabled = true,
      layout = {
        width = 42,
      },
      track = {
        enabled = true,
        width = 18,
        scroll = false,
        scroll_separator = "  ",
      },
      bar = {
        enabled = true,
        style = "braille",
        width = 10,
      },
      time = {
        enabled = true,
      },
      refresh = {
        interval_ms = 500,
      },
      component = {
        frame = {
          enabled = false,
          left = "",
          right = "",
          padding = " ",
        },
        -- separator = { left = "", right = "" },
        -- padding = { left = 0, right = 0 },
      },
      highlight = {
        default = {
          fg = "#ffffff",
          bg = "#5b7ee5",
          gui = "bold",
        },
        states = {
          playing = { bg = "#5b7ee5" },
          interval = { bg = "#7c6f64" },
          stopped = { bg = "#6c7086" },
          error = { bg = "#f38ba8" },
        },
      },
    },
  })
<

Toggle it with:
>
  :AmbientProgressToggle
<

Built-in styles:
>
  braille     sparse, borderless
  block       dense block progress
  line        thin line progress
  dots        dot progress
  squares     square progress
  diamonds    diamond progress
  pipes       vertical bar progress
  ascii       ASCII bracketed progress
  brackets    classic bracketed block progress
  angle       angle-wrapped line progress
  powerline   progress-bar wrapper using 
  separators  progress-bar wrapper using 
  rounded     progress-bar wrapper using  and 
  slanted     progress-bar wrapper using  and 
<

Aliases include:
>
  default, sparse -> braille
  blocks, dense -> block
  classic, old, bracket -> brackets
  separator, segment -> separators
  bubble, round -> rounded
<

Styles only change the progress bar characters and the progress bar's own
wrapper. `progress.component.frame` remains an explicit text-frame setting.

To match the rest of lualine, leave `progress.component.separator` and
`progress.component.padding` unset. Set them only when ambient should override
lualine's global component separators or padding.

`progress.component.separator` uses lualine's transition separator mechanism.
It may be invisible when both sides have the same background. For a separator
that should always be visible, use `progress.component.frame.left/right`.

If you do not use lualine, call `require("ambient").statusline()` from your own
statusline.

CONFIGURATION                                             *ambient-config*

Default-shaped configuration:
>
  require("ambient").setup({
    enable = true,
    music_dir = "~/.config/nvim/ambient.music",
    music_dirs = nil,
    playlists = nil,
    extensions = { "mp3", "ogg", "flac", "wav", "m4a", "aac", "opus" },
    recursive_depth = 4,
    mode = "interval_random",
    volume = 50,
    interval = {
      min_ms = 2 * 60 * 1000,
      max_ms = 5 * 60 * 1000,
    },
    progress = {
      enabled = true,
      layout = {
        width = 42,
      },
      track = {
        enabled = true,
        width = 18,
        scroll = false,
        scroll_separator = "  ",
      },
      bar = {
        enabled = true,
        style = "braille",
        width = 10,
        filled = "⠶",
        empty = "⠄",
        left = "",
        right = "",
      },
      time = {
        enabled = true,
      },
      refresh = {
        interval_ms = 500,
      },
      component = {
        frame = {
          enabled = false,
          left = "",
          right = "",
          padding = " ",
        },
        -- separator = { left = "", right = "" },
        -- padding = { left = 0, right = 0 },
      },
      highlight = {
        default = {
          fg = "#ffffff",
          bg = "#5b7ee5",
          gui = "bold",
        },
        states = {
          playing = { bg = "#5b7ee5" },
          interval = { bg = "#7c6f64" },
          stopped = { bg = "#6c7086" },
          error = { bg = "#f38ba8" },
        },
      },
    },
    show_notifications = true,
    show_notification = {
      disable_all = false,
      when_finish_setup = true,
      when_show_total_music_count = true,
      when_start_playing = true,
      when_toggle_playing_state = true,
    },
  })
<

Important options:

enable
    Set to false to disable playback through |:AmbientStart|.

music_dir
    Single music directory.

music_dirs
    List of music directories. If set, replaces music_dir.

playlists
    Full playlist definitions. If set, replaces music_dir and music_dirs.

extensions
    Default file extensions for directory-based playlists.

recursive_depth
    Default recursion depth for directory scanning.

mode
    Playback mode. See |ambient-modes|.

volume
    Value passed to `mpv --volume`. Valid range: 0 to 100.

interval.min_ms, interval.max_ms
    Random silent interval bounds, in milliseconds.

progress.enabled
    Whether the statusline progress component starts visible.

progress.layout.width
    Fixed display width for ambient's statusline text. lualine separators and
    padding are outside this width.

progress.track.enabled
    Whether to show the current track title.

progress.track.width
    Maximum displayed width for the current track name.

progress.track.scroll
    Whether long track names scroll instead of being truncated.

progress.track.scroll_separator
    Text inserted between repeated track names while scrolling.

progress.bar.style
    Built-in visual preset. Supported styles are: braille, block, line, dots,
    squares, diamonds, pipes, ascii, brackets, angle, powerline, separators,
    rounded, and slanted. Aliases include default, sparse, blocks, dense,
    classic, old, bracket, separator, segment, bubble, and round.

progress.bar.enabled
    Whether to show the progress bar and percentage.

progress.bar.width
    Width of the text progress bar.

progress.bar
    Characters used to render the fixed-width progress bar. `filled` and
    `empty` must each occupy exactly one display cell. `left` and `right`
    wrap only the progress bar, not the whole statusline component.

progress.time.enabled
    Whether to show elapsed and total time, for example 00:01/00:12.

progress.refresh.interval_ms
    Statusline refresh interval.

progress.component.frame.enabled
    Whether to render the text frame inside the fixed-width component.

progress.component.frame.left, progress.component.frame.right
    Left and right text frame strings. The total progress.layout.width includes
    these strings.

progress.component.frame.padding
    Text inserted between the frame and the content.

progress.component.separator
    lualine component separator override. Omit this option to inherit lualine's
    global component separators. Set `left` and `right` when ambient should use
    a different separator.

progress.component.padding
    lualine component padding override. Omit this option to inherit lualine's
    default padding. Set a number or `{ left = N, right = N }` for an
    ambient-specific override.

progress.highlight.default
    Default lualine color table. Supports `fg`, `bg`, and `gui`.

progress.highlight.states
    Per-state lualine color overrides. Supported keys include default, ready,
    playing, interval, stopped, paused, next, and error.

show_notifications
    Boolean shorthand. Set to false to disable all notifications.

show_notification
    Fine-grained notification switches.

LUA API                                                     *ambient-api*

require("ambient").setup(opts)                         *ambient.setup()*
    Configure ambient.nvim and scan playlists.

require("ambient").start()                             *ambient.start()*
    Start scheduling and playback.

require("ambient").stop()                              *ambient.stop()*
    Stop scheduling and playback.

require("ambient").pause()                             *ambient.pause()*
    Pause the current track.

require("ambient").toggle_pause_resume()               *ambient.toggle_pause_resume()*
    Pause while playing, resume while paused, or start playback immediately.
    During a quiet interval, start the next track immediately.

require("ambient").toggle_start_stop()                 *ambient.toggle_start_stop()*
    Toggle between active playback/scheduling and stopped. A paused track is
    considered active and is stopped.

require("ambient").next()                              *ambient.next()*
    Play the next track immediately.

require("ambient").previous()                          *ambient.previous()*
    Play the previous track from playback history.

require("ambient").select_playlist(index)              *ambient.select_playlist()*
    Select the active playlist by its one-based index.

require("ambient").select_playlist_ui()                *ambient.select_playlist_ui()*
    Open |vim.ui.select()| to choose the active playlist.

require("ambient").select_music_item()                 *ambient.select_music_item()*
    Choose a display sort order, then select and immediately play a track from
    the active playlist with |vim.ui.select()|.

require("ambient").status()                            *ambient.status()*
    Return the scheduler status result.

require("ambient").toggle_progress()                   *ambient.toggle_progress()*
    Toggle the statusline progress component.

require("ambient").show_progress()                     *ambient.show_progress()*
    Show the statusline progress component.

require("ambient").hide_progress()                     *ambient.hide_progress()*
    Hide the statusline progress component.

require("ambient").statusline()                        *ambient.statusline()*
    Return the text rendered by the statusline progress component.

  :checkhealth ambient

- current mode and volume after setup
- whether configured music directories exist

==============================================================================
TROUBLESHOOTING                                 *ambient-troubleshooting*

No sound:
    Confirm that `mpv` is installed and your system audio output works. Also
    check that `volume` is not 0.

No duration or progress percentage:
    Install `ffprobe`. Playback still works without it, but the plugin cannot
    read exact track duration.

No statusline progress:
    Confirm that `progress.enabled = true` and that lualine.nvim is installed.
    Without lualine, add `require("ambient").statusline()` to your own
    statusline.

No tracks found:
    Check `music_dir`, `music_dirs`, or `playlists`, and verify that the file
    extension is listed in `extensions` or the playlist's `ext`.
